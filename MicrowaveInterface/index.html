Existing Interface Example

 Interface is a poor design because timer does not clearly show how much time is remaining for the cook and power only has two settings. Also, when setting a cook time the user does not have great control to set a desired exact cook time.

Design Flaws:

Most Common Uses:

setting cook time
adjusting cook time

Less Common Uses:

adjusting cook power

Sequence of Actions:

open door
place food inside microwave
close door
set cook time
start

User Interaction Support:

 This microwave does not include any feedback to help the user understand how to operate the microwave. The only information given is a suggested cook time for various types of food.

User Feedback:

 This microwave only gives feedback to the user when the cook time has expired. The feedback is given in the form of a single 'ding'.

Common Mistakes:

 The microwave only works when the microwave door is closed. However, this design does not give the user feedback if the user is trying to start cooking when the door is open.

Suggested Improvements:

1: Allow the user to input an exact cook time more easily. This can be done in the form of entering the time using buttons on a number pad.

2: Assist the user on finding out what to do when the trying to start cooking when the microwave door is open.

3: Provide more feedback to the user when interacting with the microwave and when the cook time has expired.

4: Offer more easily visible and exact information regarding how much time is remaining for a cook in progress.
